math_fmod 解决实际应用问题

  • Posted on: 2015-11-19
  • By: mugbya
  • With:

缘起

记录这篇并不是突然心血来潮，而是确实在实际中遇到了，并且用数学的方式快捷简单的搞定问题。

需求

在实际项目中某字段是float型，需求保留一位小数，但要求如下:

2.01   ---->  2.0
2.499  ---->  2.0
2.501  ---->  2.5
2.990  ---->  2.5

解决

咋一看，有点向下取数的感觉，但是不能用 floor(保留的是整数)。 而round 是跟整数部分奇偶性有关的四舍五入，用起来很麻烦。

这样看来是没有系统api可以调用了，就想到自己写逻辑实现。

自己确实写了一个最容易想到的，毕竟太年轻。但是群里面有人用了数学思维解决了这个需求。

代码大概如下： 

 def cal_num(num):
return num - math.fmod(num, 0.5)

cal_num(2.01)    ---> 2.0
cal_num(2.499)   ---> 2.0
cal_num(2.501)   ---> 2.5
cal_num(2.990)   ---> 2.5

math.fmod 的描述如下：

math.fmod(x, y)
Return fmod(x, y), as defined by the platform C library.注意：Python 表达式 x % y 可能不会返回相同结果。C 标准意图，譬如：函数 fmod(x, y) 数学上会精确 (到无穷精度) 等于表达式 x - n*y (其中 n 是整数)；该函数结果具有如 x 的相同正负号且幅度小于 abs(y) 内置函数。相反，Python 操作 x % y 会返回具有 y 正负号的结果，且不会精确计算浮点数自变量。譬如：函数 fmod(-1e-100, 1e100) 的结果是 -1e-100，而 Python 操作 -1e-100 % 1e100 的结果是 1e100-1e-100 (不能精确表示浮点数结果，还会舍入成出人意料的 1e100)。由于这种原因，当处理浮点数时，通常首选 fmod() 函数；当处理整数时，首选 Python 操作 x % y。

理解这个真的是跟数学思维有关的问题了，所以也就只有写到如此，学习理解这种思维就好。
