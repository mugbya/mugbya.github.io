openerp下关联字段如何用domain做过滤

  • Posted on: 2015-12-8
  • By: mugbya
  • With:

对于openerp 来说，关联字段（如many2one型）做domain 并不是难事。只需根据条件进行过滤即可。这里拿省市级联选来说，根据所选的省份动态列出其下所有的市。

这里有一个很明显的外键关系。市对应的表中维护了一个省的外键。 这样我们就可以对市这个字段做过滤。

形如：

<field name="work_state"/>
<field name="work_city" domain="[('state_id', '=', work_state)]"/>

state_id 是work_city所对应表(即市表)的外键字段

上面的例子是 domain 的典型应用。而下面会将一个别样的例子

某些时候，当你要需要用domain过滤的字段所在的表没有那么多可以有效用来设置条件的值时，那么上面那种方式就不好使了。

这时我们用普通字段是过滤不出来的，只能通过它的主键过滤，但是要想过滤主键却在xml 中的domain是不好写的，它是个动态获取值的集合。

所以需要另辟溪路了。在了解到 on_change 的返回值可以返回一个domain 后，那么我们就可以进行下一步了。

获取过滤的集合值跟怎么返回domain 都在下面：

   #  获得准假记录中的休假类型
   holiday_status_id_list = [allot['holiday_status_id'][0] for allot in self.pool.get('hr.holidays.allot').search_read(cr, uid, domain=[('employee_id', '=', employee_id)])]

   # 获得休假规则中的休假类型
   status_record_ids = self.pool.get('hr.holidays.status').search(cr, uid, [('hr_holidays_rule_package_id', '=', employee_obj.hr_holidays_package_id.id)])

   # 休假规则下的准假记录中的休假类型
   status_id_list = list(set(holiday_status_id_list).intersection(set(status_record_ids)))

return {'value': vals,
        'domain':{'holiday_status_id': [('id','in', status_id_list)]}}

在给出的链接中提到 on_change 可以返回三种类型，其中value 是最常用的，自然不必说。 domain 在本文已有介绍。 而最后一种返回 warning 。链接中的写法是有问题的。正确的写法如下：

return {'warning':{'title': '警告', 'message': u'请不要手动对该值做修改!'}}
