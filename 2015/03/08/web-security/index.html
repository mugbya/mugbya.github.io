读<白帽子讲web安全>

  • Posted on: 2015-03-8
  • By: mugbya
  • With:

前言

我不知道喜欢是什么,或者准确的说是:我不知道怎样来描述.我喜欢的,是我的眼见而触发心中有感而有.不知道以后会喜欢什么,会不会改变?但是目前我将要说的下文,就是我从高三开始逐渐醒悟,知道自己有所喜欢的一件事.
喜欢这么久,却还是这样,看来自己还真是太无为了.

如今有人指点,也有时间,我喜欢它的激情又澎湃起来了,我想在这件事情上有所作为.

感受

读了这本书的第一章,真的算是开了一下眼界. 作者对安全的思考的角度让我学习了,里面有多东西都让我感觉到新奇,而且感觉很不错. 我甚至觉得用之来写论文或者文档都是很好的借鉴材料.比如提到的下面一些东西:

  • 安全问题的本质是信任的问题

一切的安全方案设计的基础，都是建立在信任关系上的。我们必须相信一些东西，必须有一些最基本的假设，安全方案才能得以建立；如果我们否定一切，安全方案就会如无源之水，无根之木，无法设计，也无法完成。…从另一个角度来说，一旦我们作为决策依据的条件被打破、被绕过，那么就会导致安全假设的前提条件不再可靠，变成一个伪命题。因此，把握住信任条件的度，使其恰到好处，正是设计安全方案的难点所在，也是安全这门学问的艺术魅力所在。

  • 安全的核心是数据安全问题

文章从第二章其就开始介绍详细的安全的知识,但自己基础太差,很少能完全弄懂的.像js不能控制http协议头,我就完全没有这个概念,这是什么,怎么不能?思考更多发现自己更是不足;在更后面的章节,自己能懂的就更少了,感到看下去确实吃力些,不过一次看不懂,接着再看,总能慢慢看懂.给自己加油
