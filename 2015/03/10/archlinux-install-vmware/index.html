archlinux下安装vmware

  • Posted on: 2015-03-10
  • By: mugbya
  • With: 
env: Arch 3.18.6-1-ARCH ; vmware-workstation : 11.1.0.2496824

缘起

在archlinux下我用的虚拟机都是vbox,但是这次因为接受一个vmware虚拟的系统,我还使劲力气都不能将格式转换成vbox支持的格式.无法,只能在arch上装vmware.

参考链接

  • vmware-archwiki(English)
  • vmware-archwiki(中文)
  • Linux系统上安装Vmware Workstation
  • linux下安装VMware出错：Gtk-Message: Failed to load modul

须知

archlinux默认没有locate指令,所以需要先自行安装.

# pacman -S mlocate

安装完,执行如果出现下面错误

# locate rc.conf
locate: can not stat () `/var/lib/mlocate/mlocate.db': No such file or directory

则请执行:

# updatedb

vmware的卸载指令是: vmware-installer -u vmware-workstation
vmware 11 的序列码: 1F04Z-6D111-7Z029-AV0Q4-3AEH8

问题罗列

module 丢失

error:

Gtk-Message: Failed to load module "canberra-gtk-module": libcanberra-gtk-module.so: 无法打开共享对象文件: 没有那个文件或目录

先查找libcanberra-gtk-module.so,如下是我在本地找到的; 但是如没有找到请安装 extra/libcanberra

➜  ~  locate libcanberra-gtk-module.so
/usr/lib/gtk-2.0/modules/libcanberra-gtk-module.so
/usr/lib/gtk-3.0/modules/libcanberra-gtk-module.so

添加系统加载modules的路径。

# vim /etc/ld.so.conf.d/gtk-2.0.conf
/usr/lib/gtk-2.0/modules

重新加载后重装。

# ldconfig

内核不匹配

error:

before you can run vmware several modules must be compiled and loaded into the running kernel

看到这个,你需要编译头文件,让vmware能用编译后与之匹配的内核

vmware-modconfig --console --install-all

再次之前一定要确认你有 linxu-headers,如果出现 c header files matching your running kernel were not found ,说明你该去自己安装与之匹配的头文件了
