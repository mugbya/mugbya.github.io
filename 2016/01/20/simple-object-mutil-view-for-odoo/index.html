odoo 下单对象对应多视图

  • Posted on: 2016-01-20
  • By: mugbya
  • With:

odoo 单对象如果要展示 tree 跟 form 视图的，只需在 action 中 指定 tree 跟 form 就ok

但某些需求 需要 单对象 对应不同的视图显示时，那odoo 下能怎么实现呢

需求

单对象除了表示业务外，还要在其上体现工作流。 但是在展示时有要求，处于待审批，跟已审批 不需要odoo 自带的 一些按钮（如： 新建， 删除， 保存，编辑 ，放弃）。
但是 这些内置按钮却不能通过 流程流装状态来动态控制是否展示（是不能吧，如果能，请大神指点我）。 这里就有一种实现方法是用单对象对应多个 视图。

实现

对单对象写不同form 、 tree 视图， 其中一些给新建，编辑权限，一些去掉这些权限。

指定多个视图后，那么action 在找视图时就需要显示指定那个action 对应哪个 form 视图跟tree视图。

如：

默认版

<record model="ir.ui.view" id="view_holiday_allocation_tree">
    <field name="name">hr.holidays.allocation.tree</field>
    <field name="model">hr.holidays</field>
    <field name="arch" type="xml">
        <tree colors="red:state == 'refuse';blue:state == 'draft';black:state in ('confirm','validate','validate1')" >
            <field name="employee_id"/>
        </tree>
    </field>
</record>

阉割版

<record model="ir.ui.view" id="disable_view_holiday_allocation_tree">
    <field name="name">hr.holidays.allocation.tree</field>
    <field name="model">hr.holidays</field>
    <field name="arch" type="xml">
        <tree create='false' delete='false' edit='false' colors="red:state == 'refuse';blue:state == 'draft';black:state in ('confirm','validate','validate1')" >
            <field name="employee_id"/>
        </tree>
    </field>
</record>

form 同理。

在action 中关联自己需要的view 。 注意 view_id 跟 context

<record model="ir.actions.act_window" id="hr_holidays_management.request_approve_holidays">
    <field name="name">待审批休假单</field>
    <field name="res_model">hr.holidays</field>
    <field name="view_type">form</field>
    <field name="view_mode">tree,calendar,form</field>
    <field name="view_id" ref="hr_holidays_management.disable_view_holiday_allocation_tree"/>
    <field name="context">{'form_view_ref': 'hr_holidays_management.disable_edit_holiday_new'}</field>
    <field name="domain">[('approver_ids','=',uid)]</field>
</record>

这样我们实现了 单对象对应不同的视图。

其他

当一个对象有多个视图时，还可以通过 优先级还控制指定哪个。

还有通过 ir.actions.act_window.view 实现 。

<record model="ir.actions.act_window" id="action_my_hr_employee_seq">
    <field name="name">Angajati</field>
    <field name="res_model">hr.employee</field>
    <field name="view_type">form</field>
    <field name="view_mode">tree,form</field>
</record>

<record model="ir.actions.act_window.view" id="act_hr_employee_tree_view">
    <field eval="1" name="sequence"/>
    <field name="view_mode">tree</field>
    <field name="view_id" ref="your_tree_view_id"/>
    <field name="act_window_id" ref="action_my_hr_employee_seq"/>
</record>

<record model="ir.actions.act_window.view" id="act_hr_employee_form_view">
    <field eval="2" name="sequence"/>
    <field name="view_mode">form</field>
    <field name="view_id" ref="your_form_view_id"/>
    <field name="act_window_id" ref="action_my_hr_employee_seq"/>
</record>
