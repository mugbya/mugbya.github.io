简介

PhantomJS

PhantomJS是一个无界面的,包含了WebKit浏览器引擎和JavaScript API的脚本解释器.虽然没有界面，但dom渲染、js运行、网络访问、canvas/svg绘制等功能都很完备，在页面抓取、页面输出、自动化测试等方面有广泛的应用。

casperjs

而casperjs简化PhantomJS脚本编写尤其是导航操作脚本的轻量级库.CasperJS是一个开源的,用JavaScript编写的,基于PhantomJS的导航脚本和测试工具 ,它简化了定义一个完成的导航操作所需的步骤,还提供了很有用的函数封装,方法,和语法糖

Evaluate”不工作“

Evaluate doesnt work 这是摘自stackOverflow的一个问题。

说是不能工作，其实是指不能在evaluate函数中不能执行console.log()语句。这个东西我试出来过，但是没深究，今天正好碰到了，就记录下来。

PhantomJS won’t log console messages in .evaluate() statements by default. Just include

page.onConsoleMessage = function (msg) {
     console.log(msg);
};

Evaluate与NodeList

查找dom节点，有时会用到这个querySelectorAll。它会返回一个NodeList对象，但是我们的evaluate函数却并不能很好的处理它。至于问题现象-你能够参考这里：questions

引用下别人给的解释：

PhantomJS evaluate() cannot serialize and return complex objects like HTMLElements or NodeLists, so you have to map them to serializable things before:

code:

var links = page.evaluate(function() {
    return [].map.call(document.querySelectorAll('a.listing-thumb'), function(link) {
        return link.getAttribute('href');
    });
});

打开多个链接

如代码所示，我在首页中截取出来一个链接的数组，并要打开每一个链接，在这里就需要注意了

  • 打开多个链接，需要创建不同的实例，一个实例不能同时维持多个页面的链接，其实这个好理解，跟客户端浏览器一样样的，所以我在for循环时，为每一个链接创建了一个实例

要同时打开多个链接，则要创建多个phantom实例对象，这些对象在for循环中一旦执行到sonPage.open()函数时，则闭包的情况出现。如果按照一般的java代码写法，会出现问题，如下代码所示：

for(var i = 0; i < links.length; i++){
    var sonPage = require('webpage').create();
    sonPage.open(links[i], function(){
        console.log(i);
    });
}

当你跑起来时会看到这里的输出全都是数组长度的值。这是我们需要写成正确的闭包形式。现在改成成下面的写法

for(var i = 0; i < links.length; i++){
    var sonPage = require('webpage').create();
    sonPage.open(links[i], (function(i){
      return function(){
        console.log(i);
      };
    })(i));
}

则问题解决。
